# Scoring Agent Design

HR Search Agent의 Scoring Agent는 후보 추천 순위를 사람이 이해할 수 있는 점수 체계로 재정렬한다.

Claude는 점수를 계산하지 않는다. 점수 계산은 Python에서 재현 가능하게 수행하고, Claude는 산출된 점수와 근거를 자연어로 설명한다.

## 1. 전체 원칙

최종Score는 질의에 실제로 적용된 평가항목 기준으로 정규화한다.

```text
최종Score = 획득점수 / 적용가능점수 * 100
```

항목별 기본 구조:

```text
직무적합도: 항상 50점
경험깊이: 항상 20점
리더십: 질의에 리더십 조건이 있을 때만 15점
이동/배치: 질의에 이동/파견/해외 조건이 있을 때만 10점
```

예:

```text
리더십/이동 조건 없는 질의
  적용가능점수 = 70점

리더십 조건 있는 질의
  적용가능점수 = 85점

리더십 + 이동 조건 있는 질의
  적용가능점수 = 95점
```

## 2. 직무적합도 50점

임베딩 유사도를 전체 구성원 분포 기준 Z-score로 변환한 뒤 백분위로 표시하고, 50점 스케일로 환산한다.

계산:

```text
mean = 전체 유사도 평균
std = 전체 유사도 표준편차
z = (개인 유사도 - mean) / std
percentile = normal_cdf(z) * 100
직무적합도 = percentile * 0.5
```

정규분포 CDF는 scipy 없이 표준 라이브러리로 계산한다.

```python
normal_cdf = 0.5 * (1 + math.erf(z / math.sqrt(2)))
```

예외:

```text
std == 0이면 percentile = 50, 직무적합도 = 25
```

## 3. 경험깊이 20점

경험깊이는 경력요건충족도 10점과 주요업무깊이 10점으로 나눈다.

### 3.1 경력요건충족도 10점

질의에 경력 최소 조건이 있는 경우:

```text
경력요건충족도 = min(경력연수 / 요구연수, 1.0) * 10
```

예:

```text
요구 10년, 후보 5년  -> 5점
요구 10년, 후보 8년  -> 8점
요구 10년, 후보 10년 -> 10점
요구 10년, 후보 20년 -> 10점
```

질의에 경력 최소 조건이 없는 경우:

```text
경력요건충족도 = 7점
```

경력 조건이 없는 질의에서 경력연수 percentile을 강하게 반영하면 사용자가 요구하지 않은 "연차"가 순위를 과도하게 좌우할 수 있다. 따라서 이 경우에는 중립 기본점 7점을 부여하고, 직무적합도와 주요업무깊이가 순위에 더 크게 작동하도록 한다.

### 3.2 주요업무깊이 10점

주요업무 1~3의 비중, 난이도, 대체불가성을 점수화한다.

난이도:

```text
상=3, 중=2, 하=1
```

대체가능성:

```text
하=3, 중=2, 상=1
```

업무별 점수:

```text
비중(%) * 난이도 * 대체불가성
```

개인 업무깊이:

```text
업무1점수 + 업무2점수 + 업무3점수
```

전체 30명 기준 percentile로 변환:

```text
주요업무깊이 = percentile * 10
```

## 4. 리더십 15점

리더십 조건은 질의에 리더십이 명시된 경우에만 최종점수에 반영한다.

적용 조건:

```text
hard_conditions["팀리딩필요"] is not None
또는 역량키워드에 "리더십", "팀리딩", "조직관리" 등 포함
```

계산:

```text
팀리딩여부: 7점
팀리딩인원 percentile: 최대 4점
팀리딩기간_월 percentile: 최대 4점
```

팀리딩여부가 false이면 전체 리더십 점수는 0점이다.

## 5. 이동/배치 10점

이동/배치 조건은 질의에 관련 조건이 명시된 경우에만 최종점수에 반영한다.

적용 조건:

```text
이동가능시점 조건 있음
파견가능 조건 있음
해외근무가능 조건 있음
```

계산:

```text
이동가능시점: 최대 4점
파견가능여부: 최대 3점
해외근무가능: 최대 3점
```

이동가능시점 점수:

```text
요구 즉시:
  즉시=4, 그 외=0

요구 3개월내:
  즉시=4, 3개월내=3, 그 외=0

요구 6개월내:
  즉시=4, 3개월내=3, 6개월내=2, 그 외=0

요구 1년내:
  즉시=4, 3개월내=3, 6개월내=2, 1년내=1
```

파견/해외 조건은 요구 조건과 후보 값이 일치하면 만점, 불일치하면 0점이다.

## 6. Percentile 계산

동점 처리를 위해 `개인보다 낮은 값의 수`와 `같은 값의 수`를 함께 사용한다.

```text
percentile = (lower_count + 0.5 * equal_count) / n
```

0~1 범위로 계산한 뒤 항목별 만점에 곱한다.

전체 인원이 1명인 경우 percentile은 1.0으로 처리한다.

## 7. 출력 구조

Scoring Agent 출력:

```json
{
  "scored_candidates": [
    {
      "순위": 1,
      "사번": "2005001",
      "성명": "김민준",
      "소속조직": "재무팀",
      "최종Score": 92.3,
      "획득점수": 78.5,
      "적용가능점수": 85,
      "점수상세": {
        "직무적합도": {},
        "경험깊이": {},
        "리더십": {},
        "이동배치": {}
      },
      "Score산출근거": "..."
    }
  ],
  "original_query": "...",
  "score_policy": {
    "직무적합도": 50,
    "경험깊이": 20,
    "리더십": 15,
    "이동배치": 0
  }
}
```

## 8. UI 표시 원칙

UI에는 raw cosine similarity를 크게 보여주지 않는다.

우선 표시:

```text
최종Score
직무적합도
경험깊이
리더십
이동/배치
```

상세 펼침에서만 표시:

```text
raw cosine similarity
z-score
percentile
산출과정
```
